# Security Policy

## Reporting a vulnerability

Please report security or safety-sensitive issues privately by email:

```text
komoootv@gmail.com
```

Include:

- the affected file, skill, script, or plugin metadata;
- a short reproduction or scenario;
- the expected impact;
- whether the issue can cause data exposure, unsafe tool use, command execution risk, or misleading research claims.

Please do not open a public issue for vulnerabilities until the report has been triaged.

## Scope

This repository contains local skill instructions, plugin metadata, Markdown templates, and Python utility scripts. Security reports are most useful when they identify an externally observable failure mode, such as unsafe command guidance, unsafe artifact handling, path traversal, misleading validation, or a plugin metadata problem that affects installation or use.

## Supported versions

The `main` branch is the supported development line. Release metadata is tracked in the plugin manifests.
